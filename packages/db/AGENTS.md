# packages/db

**Scope**: Database layer - Drizzle ORM + PostgreSQL

## Overview

Type-safe database client with Drizzle ORM, PostgreSQL via Neon, schema definitions, and migrations.

## Structure

```
packages/db/
├── src/
│   ├── schema/          # Table definitions
│   │   ├── index.ts     # Schema exports
│   │   ├── auth.ts      # Better-Auth tables (user, session, account, verification)
│   │   ├── subscriptions.ts  # Subscription records (Stripe)
│   │   └── generated-content.ts  # AI-generated content history
│   ├── migrations/      # Drizzle migrations
│   │   ├── meta/        # Migration metadata
│   │   └── *.sql         # Migration files
│   ├── queries/         # Query helpers
│   │   ├── index.ts     # Query exports
│   │   ├── users.ts     # User queries (points, stripeCustomerId)
│   │   ├── subscriptions.ts  # Subscription queries
│   │   └── generated-content.ts  # Content history queries
│   └── index.ts         # DB client export
└── drizzle.config.ts    # Drizzle configuration
```

## Where to Look

| Task               | Location                                                            |
| ------------------ | ------------------------------------------------------------------- |
| Add table          | `src/schema/*.ts` (auth.ts, subscriptions.ts, generated-content.ts) |
| Add queries        | `src/queries/*.ts`                                                  |
| Run migrations     | `bun run db:migrate` (from root)                                    |
| Generate migration | `bun run db:generate`                                               |
| DB client          | `src/index.ts` - exports `db`                                       |
| Queries            | `src/queries/` - custom query helpers                               |

## Conventions

- **Schema files**: One file per domain (auth.ts, subscriptions.ts, generated-content.ts, todos.ts)
- **User table**: Extended from Better-Auth's user table with `points` and `stripeCustomerId` additionalFields
- **Naming**: snake_case in DB, camelCase in TypeScript
- **Relations**: Define in schema files using Drizzle relations
- **Migrations**: Auto-generated by drizzle-kit, committed to repo
- **ID types**: Better-Auth uses text IDs (UUIDs), not serial integers
- **Write helpers own ID generation**: New rows in `generated_content` and `subscriptions` generate `id` in the query helper with `crypto.randomUUID()`

## Anti-Patterns

- **NEVER edit** migration files after generation
- **No raw SQL** - use Drizzle query builder
- **No DB URL in code** - use env vars via `@modern-web-starter/env`

## Commands

```bash
# From root:
bun run db:generate   # Generate migration from schema changes
bun run db:migrate    # Run pending migrations
bun run db:studio     # Drizzle Studio UI
bun run test:db       # Run DB-focused Vitest suite from apps/server
```

## Using Queries

- Prefer `src/queries/*.ts` for app code instead of repeating Drizzle chains in routes or services
- Import query helpers from `@modern-web-starter/db/queries/...` or `@modern-web-starter/db/queries`
- Use schema imports only when you are adding a new query or changing table definitions
- Treat all Better-Auth user IDs as `string`
- Treat `generated_content.id` and `subscriptions.id` as `string`
- Do not assume serial or numeric IDs anywhere in new code

### Current query helpers

- `users.ts`
  - `getUserByStripeCustomerId(stripeCustomerId)`
  - `getUserPoints(userId)`
  - `updateUserPoints(userId, pointsToAdd)`
- `subscriptions.ts`
  - `getSubscriptionByStripeId(stripeSubscriptionId)`
  - `createOrUpdateSubscription(userId, stripeSubscriptionId, plan, status, currentPeriodStart, currentPeriodEnd)`
- `generated-content.ts`
  - `saveGeneratedContent(userId, content, prompt, contentType)`
  - `getGeneratedContentHistory(userId, limit?)`
  - `deleteGeneratedContent(id)`
- `todos.ts`
  - `insertTodo(todo)`
  - `getTodoById(id)`
  - `getTodosByUserId(userId)`
  - `updateTodo(id, data)`
  - `deleteTodo(id)`

### Usage examples

```ts
import {
  getGeneratedContentHistory,
  saveGeneratedContent,
} from "@modern-web-starter/db/queries/generated-content";

const saved = await saveGeneratedContent(user.id, content, prompt, "thread");

const history = await getGeneratedContentHistory(user.id, 20);
```

```ts
import {
  createOrUpdateSubscription,
  getSubscriptionByStripeId,
} from "@modern-web-starter/db/queries/subscriptions";

await createOrUpdateSubscription(
  user.id,
  stripeSubscriptionId,
  "pro",
  "active",
  currentPeriodStart,
  currentPeriodEnd,
);

const subscription = await getSubscriptionByStripeId(stripeSubscriptionId);
```

```ts
import { getUserPoints, updateUserPoints } from "@modern-web-starter/db/queries/users";

const points = await getUserPoints(user.id);
const updatedUser = await updateUserPoints(user.id, -5);
```

### Query rules

- Keep route handlers thin; move reusable DB logic into `src/queries`
- If a helper creates a row with a text primary key, generate the `id` inside the helper unless the caller has a stronger domain reason to supply it
- Keep return values simple and explicit; return inserted or updated rows when the caller needs immediate state
- Preserve migration-safe behavior when porting old Next.js code: old numeric IDs from legacy tables do not apply to Better-Auth-backed tables
- Add or update tests in `apps/server/src/tests/db/` whenever query behavior changes
- If changing schema defaults or ID behavior, update both the migration and the query tests in the same change

## Notes

- PostgreSQL via Neon (serverless)
- Drizzle config: `casing: 'snake_case'`
- Migrations stored in `src/migrations/`
- Better-Auth uses drizzle adapter from `@better-auth/drizzle-adapter`
- **User table extended** with `points` (integer, default 50) and `stripeCustomerId` (unique text) via additionalFields
- **Subscriptions table**: Links users to Stripe subscription data
- **GeneratedContent table**: Stores AI-generated content history per user
- Query tests live in `apps/server/src/tests/db/` because Vitest is executed from the server workspace
